**ConnectivityEvent.swift — Add CodingUserInfoKey extension and conditional encoding:**

1. After the existing imports (line 3), add a `CodingUserInfoKey` extension:

```swift
extension CodingUserInfoKey {
    /// When set to `true` on a JSONEncoder's userInfo, the encoder omits
    /// latitude, longitude, and locationAccuracy from the output.
    static let omitLocation = CodingUserInfoKey(rawValue: "omitLocation")!
}
```

2. In the `encode(to:)` method (currently lines 216-234), replace the three unconditional location encodes (lines 230-232):

```swift
try container.encodeIfPresent(latitude, forKey: .latitude)
try container.encodeIfPresent(longitude, forKey: .longitude)
try container.encodeIfPresent(locationAccuracy, forKey: .locationAccuracy)
```

with a conditional block:

```swift
let omitLocation = encoder.userInfo[.omitLocation] as? Bool ?? false
if !omitLocation {
    try container.encodeIfPresent(latitude, forKey: .latitude)
    try container.encodeIfPresent(longitude, forKey: .longitude)
    try container.encodeIfPresent(locationAccuracy, forKey: .locationAccuracy)
}
```

All other fields (timestamp, eventType, pathStatus, etc.) remain unchanged. The `init(from:)` decoder is NOT modified — it still decodes all fields when present.
