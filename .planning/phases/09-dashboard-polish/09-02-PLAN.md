**B. LocationService — Conditional write inside the existing Task block** (at the END of the `Task { @MainActor in ... }` body in `locationManager(_:didUpdateLocations:)`, AFTER the existing `UserDefaults.standard.set(..., forKey: DefaultsKey.lastActiveTimestamp)` write at lines 117-120):

```swift
Task { @MainActor in
    // 1. Update ConnectivityMonitor with new location (DAT-04) — UNCHANGED
    monitor.updateLocation(...)

    // 2. Detect and log any monitoring gap (DAT-05) — UNCHANGED
    detectAndLogGap()

    // 3. Run a single connectivity probe (wake-then-probe pattern) — UNCHANGED
    await monitor.runSingleProbe()

    // 4. Update last active timestamp for next gap check — UNCHANGED
    UserDefaults.standard.set(
        Date().timeIntervalSince1970,
        forKey: DefaultsKey.lastActiveTimestamp
    )

    // 5. NEW (POLISH-01 / D-08): record a background-wake-only timestamp. This is the
    //    "is the app still alive in the background?" signal surfaced by HealthDetailSheet's
    //    live ticker. Foreground location callbacks do NOT count — they would mask the
    //    diagnostic. `applicationState` is read inside the @MainActor Task, which is
    //    where this property is safe to access.
    if UIApplication.shared.applicationState != .active {
        UserDefaults.standard.set(
            Date().timeIntervalSince1970,
            forKey: DefaultsKey.lastBackgroundWakeTimestamp
        )
    }
}
```

> `UIApplication` is in UIKit; the existing file does NOT yet import UIKit. **Add `import UIKit`** at the top of `LocationService.swift` (alongside `import CoreLocation`, `import Observation`, `import Foundation` at lines 1-3). This is the simpler of the two options in CONTEXT.md "Claude's Discretion" — querying `applicationState` directly inside the @MainActor Task block is more local than maintaining a separate `isBackground: Bool` driven by lifecycle notifications, and the two approaches produce identical timestamp semantics.

```swift
private var lastWakeText: String {
    let lastActive = UserDefaults.standard.double(forKey: "lastActiveTimestamp")
```

```swift
/// Relative-time string for the most recent BACKGROUND wake (POLISH-01 / D-10).
/// Reads `lastBackgroundWakeTimestamp` (written only by `LocationService` when
/// `applicationState != .active`). When the key is unset / zero, returns
/// "Never (no background wake yet)" so the user sees an unambiguous signal
/// that the app has not yet been woken in the background.
///
/// Examples (Date.RelativeFormatStyle, .named units, abbreviated):
///   "12 sec ago", "3 min ago", "2 hr ago"
/// The text re-renders every second because the surrounding TimelineView ticks
/// on a 1s schedule — each tick re-evaluates this property fresh.
private var lastBackgroundWakeText: String {
    let raw = UserDefaults.standard.double(forKey: "lastBackgroundWakeTimestamp")
    guard raw > 0 else { return "Never (no background wake yet)" }
    let wakeDate = Date(timeIntervalSince1970: raw)
    return wakeDate.formatted(
        .relative(presentation: .named, unitsStyle: .abbreviated)
    )
}
```
