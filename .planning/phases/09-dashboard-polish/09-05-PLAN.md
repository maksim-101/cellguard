  <action>
    Two-part fix. Apply both because each addresses a different failure mode (horizontal truncation of long relative-time strings AND vertical clipping at the sheet's bottom edge).

    **Part A — Allow the relative-time Text to wrap to a second line if needed.**

    Inside the TimelineView closure at lines 110-119, change the layout from a horizontal HStack (which forces single-line and truncates the right Text when it overflows) to a labeled VStack so long relative-time strings can wrap. Replace the entire TimelineView block with:
    ```swift
    TimelineView(.periodic(from: .now, by: 1)) { _ in
        VStack(alignment: .leading, spacing: 2) {
            Text("Last Background Wake:")
                .font(.caption)
                .foregroundStyle(.secondary)
            Text(lastBackgroundWakeText)
                .font(.caption)
                .foregroundStyle(.secondary)
                .fixedSize(horizontal: false, vertical: true)
                .multilineTextAlignment(.leading)
        }
        .frame(maxWidth: .infinity, alignment: .leading)
    }
    ```
    Why VStack instead of HStack: the existing "Cert Expires:" row above (lines 95-103) uses HStack and stays single-line because `expirationDisplayText` is short ("in 6 days"). The wake row's value can be much longer in the empty state ("Never (no background wake yet)" — 31 chars) AND when relative-time grows ("3 minutes ago" → "1 hour 23 minutes ago" → "2 days 14 hours ago"). VStack with `.fixedSize(horizontal: false, vertical: true)` is the SwiftUI-canonical way to let a Text wrap freely without truncation.

    **Part B — Default the sheet to the larger detent so the wake row is never near the bottom edge.**

    Use a `@State`-backed `PresentationDetent` selection binding so the sheet opens at `.large` by default while still allowing the user to collapse to `.medium`.

    1. Add a new `@State` property near the other `@Environment` declarations at the top of the struct (around line 8):
       ```swift
       @State private var sheetDetent: PresentationDetent = .large
       ```

    2. Change line 135 from:
       ```swift
       .presentationDetents([.medium, .large])
       ```
       to:
       ```swift
       .presentationDetents([.medium, .large], selection: $sheetDetent)
       ```

    The default starts at `.large` so all rows are visible on first open; the user can still drag down to `.medium` via the drag indicator (line 136 `.presentationDragIndicator(.visible)`) if they want to peek at the dashboard.

    Note: we do NOT use `.constant(.large)` for the selection — that would freeze the detent at `.large` and prevent the user from collapsing to `.medium` even by dragging. The `@State` binding above is the correct form because it sets the INITIAL value to `.large` while keeping the binding two-way so user drags update it.

    Do NOT:
    - Remove `.medium` from the detents array. The user requested both detents in Phase 9 plan 09-02; reducing them is a regression.
    - Switch to `.fraction(...)` detents. `.large` is the standard pattern and matches the "see all health detail" intent.
    - Add `.lineLimit(...)` anywhere. We want unlimited wrap.
    - Touch the surrounding rows (statusHeading, statusBody, degraded reasons, Start/Stop button, radio info, Cert Expires). They are working today.
    - Remove the surrounding `VStack(alignment: .leading, spacing: 4)` at line 93 — keep the existing footer container; just swap the TimelineView's inner HStack for VStack as described.
  </action>
