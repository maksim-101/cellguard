# Phase 3 — UI Design Contract

> Visual and interaction contract for the Background Lifecycle phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | SwiftUI native (iOS 26 SDK) |
| Preset | not applicable |
| Component library | SwiftUI built-in components |
| Icon library | SF Symbols 6 |
| Font | San Francisco (iOS system default) |

**Note:** This is a native iOS app. No web design system (shadcn, Tailwind, etc.) applies. All visual contracts use Apple's Human Interface Guidelines and SwiftUI semantic tokens.

---

## Spacing Scale

SwiftUI's native spacing is used throughout. Declared values for custom spacing:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Inline gaps between status dot and label |
| sm | 8pt | Vertical padding in status bar, compact list item spacing |
| md | 16pt | Default horizontal padding, section spacing |
| lg | 24pt | Section padding in health detail sheet |
| xl | 32pt | Top/bottom margins in permission prompt views |

Exceptions: None. SwiftUI default spacing is used for List, VStack, and NavigationStack unless explicitly overridden above.

---

## Typography

Phase 3 adds minimal new UI text. All typography uses SwiftUI's Dynamic Type system fonts.

| Role | SwiftUI Style | Effective Size (Default) | Weight | Usage |
|------|---------------|--------------------------|--------|-------|
| Body | `.body` | 17pt | regular (400) | Health status description text, degraded reason list items |
| Caption | `.caption` | 12pt | regular (400) | Provisioning profile expiration date, monitoring status bar (existing) |
| Caption 2 | `.caption2` | 11pt | regular (400) | Secondary metadata (gap duration, last wake time) |
| Headline | `.headline` | 17pt | semibold (600) | Health status title ("Active", "Degraded", "Paused"), event type names (existing) |

**Weights used:** regular (400) and semibold (600) only. No custom fonts.
**Line height:** System-managed by Dynamic Type. Do not override.

---

## Color

Phase 3 uses iOS semantic colors exclusively. No custom hex values.

| Role | SwiftUI Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `.background` / system default | View backgrounds, list backgrounds |
| Secondary (30%) | `.secondarySystemGroupedBackground` | Grouped list cells, health detail sheet background |
| Accent (10%) | `.green` (system) | Active monitoring indicator dot, "Active" health status text |
| Warning | `.orange` (system) | Degraded monitoring indicator dot, "Degraded" health status text |
| Destructive | `.red` (system) | Paused/stopped monitoring indicator dot, "Paused" health status text, profile expiring warning |
| Tertiary text | `.secondary` | Timestamps, caption text, provisioning expiry date |

**Accent reserved for:** Monitoring health indicator dot when status is `.active`. Never used for buttons or generic highlights in this phase.

**Monitoring health color mapping (prescriptive):**

| Health State | Indicator Color | Text Color |
|--------------|-----------------|------------|
| Active | `Color.green` | `Color.green` |
| Degraded | `Color.orange` | `Color.orange` |
| Paused | `Color.red` | `Color.red` |

**Profile expiry color logic:**
- More than 48 hours remaining: `.secondary` (normal caption text)
- 48 hours or less remaining: `Color.red` with `.bold` weight

## Component Inventory

Phase 3 adds these UI elements to the existing ContentView:

### 1. Monitoring Health Status Bar (modified existing status bar)

**Location:** Top of ContentView, replaces current simple "Monitoring Active/Stopped" bar.

**Structure:**
```
[Health Dot 8pt] [Health Label] [Spacer] [Radio Tech] [Chevron >]
```

- Health dot: 8pt circle, color from health color mapping above
- Health label: `.caption` weight, e.g. "Monitoring Active", "Monitoring Degraded", "Monitoring Paused"
- Tapping the bar opens the Health Detail Sheet
- Chevron (`chevron.right` SF Symbol) at trailing edge indicates tappable

**States:**
| State | Dot | Label | Tappable |
|-------|-----|-------|----------|
| Active, no issues | Green | "Monitoring Active" | Yes (shows detail) |
| Degraded, 1+ reasons | Orange | "Monitoring Degraded" | Yes (shows reasons) |
| Paused / not monitoring | Red | "Monitoring Paused" | Yes (shows how to resume) |

### 2. Health Detail Sheet (new)

**Trigger:** Tap on the monitoring health status bar.
**Presentation:** `.sheet` modifier, medium detent (`.presentationDetents([.medium])`).

**Content when Active:**
```
[Green dot] Monitoring Active          [X close button]

All systems operational. Background monitoring is running.

Profile Expires: Mar 31, 2026
Last Background Wake: 2 minutes ago
```

**Content when Degraded:**
```
[Orange dot] Monitoring Degraded       [X close button]

Background monitoring may miss events due to:

[!] Low Power Mode is active
    Turn off Low Power Mode in Settings > Battery.

[!] Background App Refresh is disabled
    Enable in Settings > General > Background App Refresh.

Profile Expires: Mar 31, 2026
Last Background Wake: 14 minutes ago
```

**Content when Paused:**
```
[Red dot] Monitoring Paused            [X close button]

Monitoring is not running. Tap "Start Monitoring" to begin.

[Start Monitoring]  (button, .borderedProminent style)
```

- Each degraded reason: SF Symbol `exclamationmark.triangle.fill` in orange, reason title in `.headline`, fix instruction in `.body` with `.secondary` foreground
- Profile expiry line: `.caption` text, conditionally red if within 48 hours
- Last background wake: `.caption` text showing relative time since last `lastActiveTimestamp`

### 3. Monitoring Gap Event Row (in existing event list)

**Appearance in existing List:** Same row structure as other events but with distinct visual treatment.

```
[Monitoring Gap]                       <- .headline, same as other event types
[gap icon] Monitoring was suspended    <- .caption, .secondary
[12:04 PM - 12:47 PM (43 min)]        <- .caption2, .secondary
```

- SF Symbol for gap events: `pause.circle` (consistent with "paused" concept)
- Gap duration shown as time range with total duration in parentheses

### 4. Location Permission Prompt (system-managed)

Phase 3 triggers `requestAlwaysAuthorization()`. The system presents its own permission dialog. No custom UI needed, but the `NSLocationAlwaysAndWhenInUseUsageDescription` string is part of the copywriting contract below.

## Interaction Contracts

### Health Status Bar Tap

- **Gesture:** Single tap on the entire health status bar HStack
- **Response:** Present health detail sheet with `.sheet` modifier
- **Feedback:** Standard iOS highlight (no custom tap feedback needed)

### Start/Stop Monitoring Toggle

- **Location:** Health detail sheet (when paused)
- **Style:** `.buttonStyle(.borderedProminent)` for Start, `.buttonStyle(.bordered)` with `.tint(.red)` for Stop
- **No confirmation required** for either action

### Health Detail Sheet Dismiss

- **Gesture:** Swipe down or tap X button (`.presentationDragIndicator(.visible)`)
- **Standard iOS sheet dismissal behavior**

## Accessibility Contract

- All status indicators use both color AND text (never color alone)
- Health dot colors are paired with explicit labels ("Active", "Degraded", "Paused")
- All text respects Dynamic Type scaling
- VoiceOver: Health status bar reads as single accessible element: "Monitoring {status}, tap for details"
- Degraded reasons list: each reason is a separate VoiceOver element with title + fix instruction
