  <action>
    Create CellGuard/Services/MonitoringHealthService.swift as an `@Observable final class`.

    **Nested types:**

    ```swift
    enum Health: Equatable {
        case active
        case degraded(reasons: [DegradedReason])
        case paused
    }

    enum DegradedReason: String, CaseIterable {
        case lowPowerMode = "Low Power Mode is active"
        case backgroundRefreshDisabled = "Background App Refresh is disabled"
        case locationWhenInUse = "Location set to 'While Using'"
        case locationDenied = "Location access denied"
    }
    ```

    Add a computed `fixInstruction` property on DegradedReason:
    ```swift
    var fixInstruction: String {
        switch self {
        case .lowPowerMode: "Turn off Low Power Mode in Settings > Battery."
        case .backgroundRefreshDisabled: "Enable in Settings > General > Background App Refresh."
        case .locationWhenInUse: "Change to 'Always' in Settings > CellGuard > Location."
        case .locationDenied: "Grant location access in Settings > CellGuard > Location."
        }
    }
    ```

    **Properties:**
    - `private(set) var health: Health = .paused`
    - `private var observers: [Any] = []`
    - `private var onConditionChanged: (() -> Void)?` (stored closure for notification handlers)

    **Methods:**

    `func evaluate(isMonitoring: Bool, locationAuth: CLAuthorizationStatus, backgroundRefresh: UIBackgroundRefreshStatus)`:
    - If `!isMonitoring`, set `health = .paused` and return.
    - Build `var reasons: [DegradedReason] = []`.
    - If `ProcessInfo.processInfo.isLowPowerModeEnabled`, append `.lowPowerMode`.
    - If `backgroundRefresh != .available`, append `.backgroundRefreshDisabled`.
    - If `locationAuth == .authorizedWhenInUse`, append `.locationWhenInUse`.
    - If `locationAuth == .denied || locationAuth == .restricted`, append `.locationDenied`.
    - Set `health = reasons.isEmpty ? .active : .degraded(reasons: reasons)`.

    `func startObserving(onConditionChanged: @escaping () -> Void)`:
    - Store `self.onConditionChanged = onConditionChanged`.
    - Register for `NSNotification.Name.NSProcessInfoPowerStateDidChange` via NotificationCenter. In the observer closure, call `onConditionChanged()` on the main thread via `DispatchQueue.main.async` so the caller can re-evaluate health with current inputs. This is critical for BKG-04: when Low Power Mode toggles while the app is foregrounded, the health status bar must update immediately.
    - Register for `UIApplication.backgroundRefreshStatusDidChangeNotification`. In the observer closure, call `onConditionChanged()` on the main thread via `DispatchQueue.main.async`.
    - Store observer tokens in `self.observers` for cleanup.

    The caller (Plan 03 wiring) provides the `onConditionChanged` closure that calls `evaluate()` with the current `isMonitoring`, `locationAuth`, and `backgroundRefresh` values. This keeps MonitoringHealthService decoupled from the services it does not own while still enabling real-time health updates.

    `func stopObserving()`:
    - Remove all observers from NotificationCenter.
    - Set `onConditionChanged = nil`.

    **BGAppRefreshTask scheduling (static utility):**

    Add a static method for scheduling the next refresh task:
    ```swift
    static func scheduleAppRefresh() {
        let request = BGAppRefreshTaskRequest(identifier: "com.cellguard.refresh")
        request.earliestBeginDate = Date(timeIntervalSinceNow: 15 * 60) // 15 min minimum
        do {
            try BGTaskScheduler.shared.submit(request)
        } catch {
            print("Could not schedule app refresh: \(error)")
        }
    }
    ```

    Import: Observation, Foundation, CoreLocation, UIKit, BackgroundTasks.

    **Nested Decodable struct for plist parsing:**
    ```swift
    private struct ProvisioningProfile: Decodable {
        let name: String
        let expirationDate: Date
        let creationDate: Date

        enum CodingKeys: String, CodingKey {
            case name = "Name"
            case expirationDate = "ExpirationDate"
            case creationDate = "CreationDate"
        }
    }
    ```

    **Properties:**
    - `private(set) var expirationDate: Date?` (nil on Simulator or if profile unreadable)
    - `private(set) var profileName: String?`

    **Computed properties:**
    - `var isExpiringSoon: Bool` -- returns true if expirationDate is non-nil and within 48 hours of now.
    - `var expirationDisplayText: String` -- if expirationDate is nil, return "Unknown (Simulator)". Otherwise format with `Date.FormatStyle` using `.dateTime.month().day().year()` (e.g., "Mar 31, 2026").
    - `var daysUntilExpiry: Int?` -- if expirationDate exists, return Calendar.current.dateComponents([.day], from: Date(), to: expirationDate).day.

    **Methods:**

    `func loadProfile()`:
    - Guard let path = Bundle.main.path(forResource: "embedded", ofType: "mobileprovision"), else set expirationDate = nil and return (Simulator case, Pitfall 6).
    - Read Data from file URL.
    - Convert to ASCII String.
    - Find range of "<?xml" and "</plist>" to extract the plist XML from the binary provisioning profile.
    - Decode the extracted plist substring using PropertyListDecoder into ProvisioningProfile.
    - Set `expirationDate` and `profileName` from the decoded struct.
    - Call `scheduleExpiryNotification()` if expirationDate is valid.

    `private func scheduleExpiryNotification()`:
    - Guard let expirationDate else return.
    - Get UNUserNotificationCenter.current().
    - Request notification authorization with [.alert, .sound] options (async).
    - Remove pending notifications with identifier "profileExpiry".
    - Calculate warningDate = expirationDate.addingTimeInterval(-48 * 3600).
    - Guard warningDate > Date() else return (already past warning window).
    - Create UNMutableNotificationContent with:
      - title: "CellGuard Profile Expiring"
      - body: "Your provisioning profile expires in 48 hours. Re-sign the app in Xcode to continue monitoring."
      - sound: .default
    - Create UNTimeIntervalNotificationTrigger with timeInterval: max(warningDate.timeIntervalSinceNow, 1), repeats: false.
    - Create UNNotificationRequest with identifier "profileExpiry".
    - Add request to center.

    Import: Observation, Foundation, UserNotifications.
