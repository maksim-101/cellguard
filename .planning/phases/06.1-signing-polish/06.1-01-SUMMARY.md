# Summary: Signing Polish & Warning Fixes

## What Was Built

Fixed provisioning profile parsing on iOS 26 and resolved all Xcode warnings.

## Changes Made

1. **ProvisioningProfileService.swift** — Fixed profile parsing by switching from `.ascii` to `.isoLatin1` encoding (iOS 26 profiles contain bytes > 127 in the CMS/DER wrapper). Added direct bundle path fallback and build-date estimation fallback. Added `isEstimatedExpiry` flag.
2. **EventLogExport.swift** — Marked `@unchecked Sendable`, pre-captured `deviceModel`/`osVersion` at call site to avoid async UIDevice access in transfer closure. Removed deprecated CTCarrier usage.
3. **ConnectivityMonitor.swift** — Removed deprecated CTCarrier carrier name capture (returns nil on iOS 26, no replacement exists). Radio tech still works.
4. **AppDelegate.swift** — Added `@MainActor` for concurrency safety. Removed dead `launchOptions[.location]` code (deprecated iOS 26, flag was never read).
5. **AccentColor.colorset** — Added to asset catalog.
6. **DashboardView.swift** — Updated EventLogExport call site with pre-captured device info.

## Key Files

- `CellGuard/Services/ProvisioningProfileService.swift` — isoLatin1 + fallback chain
- `CellGuard/Models/EventLogExport.swift` — Sendable + async fixes

- `CellGuard/App/AppDelegate.swift` — @MainActor + dead code removal
