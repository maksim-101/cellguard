```swift
networkInfo.serviceCurrentRadioAccessTechnologyDidUpdateNotifier = { [weak self] serviceID in
    guard let self else { return }
    let newTech = self.networkInfo.serviceCurrentRadioAccessTechnology?[serviceID]

    }
}
```

**Add `func startProbeTimer()`:**
1. `stopProbeTimer()` first (safety)
2. `probeTimer = Timer.scheduledTimer(withTimeInterval: probeInterval, repeats: true) { [weak self] _ in guard let self else { return }; Task { await self.runProbe() } }`
3. Run first probe immediately: `Task { await runProbe() }`

**Add `func stopProbeTimer()`:**
1. `probeTimer?.invalidate()`
2. `probeTimer = nil`

**Add `private func runProbe() async`:** (MON-02, MON-03)
1. Capture state at probe initiation time (Pitfall 5 -- race condition): `let capturedStatus = currentPathStatus`, `let capturedInterface = currentInterfaceType`
2. Create `var request = URLRequest(url: probeURL)`, set `request.httpMethod = "HEAD"`
3. Record `let start = Date()`
4. In do/catch:
   - **Success path:** `let (_, response) = try await probeSession.data(for: request)`. Calculate `let latencyMs = Date().timeIntervalSince(start) * 1000`. Check `(response as? HTTPURLResponse)?.statusCode == 200`. If 200: call `logEvent(type: .probeSuccess, probeLatencyMs: latencyMs)`. If not 200: call `logEvent(type: .probeFailure, probeFailureReason: "HTTP \(statusCode)")`.
   - **Failure path (catch):** Calculate latencyMs. Check if `capturedStatus == .satisfied && capturedInterface == .cellular` -- if so, this is a **silent modem failure (MON-03)**: call `logEvent(type: .silentFailure, probeLatencyMs: latencyMs, probeFailureReason: error.localizedDescription)` and set `dropStartDate = dropStartDate ?? Date()` (start tracking drop if not already). Otherwise: call `logEvent(type: .probeFailure, probeLatencyMs: latencyMs, probeFailureReason: error.localizedDescription)`.

**Modify `logEvent` helper:** Update the `radioTechnology` parameter to use `captureRadioTechnology()` instead of `nil`. Update `carrierName` to use `captureCarrierName()` instead of `nil`.

**Add `private func captureRadioTechnology() -> String?`:** (MON-04)
```swift
return networkInfo.serviceCurrentRadioAccessTechnology?.values.first
```

**Add `private func captureCarrierName() -> String?`:** (MON-05)
```swift
// Deprecated since iOS 16.4 -- may return nil on iOS 26. Best-effort per MON-05.

```

**Modify `logEvent` to pass location (DAT-04):**
Ensure `latitude: lastLocation?.latitude`, `longitude: lastLocation?.longitude`, `locationAccuracy: lastLocation?.accuracy` are passed to every ConnectivityEvent init call. The `lastLocation` is already set by `updateLocation()` from Plan 01. Phase 3 will plug in the actual CLLocationManager.

**Important notes:**
- Reuse ONE URLSession instance (`probeSession`) -- do NOT create per probe (anti-pattern from research)
- Use `waitsForConnectivity = false` -- we WANT immediate failure to detect silent modem failures
- Capture path state BEFORE awaiting probe result (Pitfall 5)
- Timer stops firing in background -- this is correct for Phase 2 (foreground only). Add comment noting Phase 3 adds background wake-then-probe.

Replace the current minimal app entry with full ConnectivityMonitor wiring:

```swift
import SwiftUI
import SwiftData

@main
struct CellGuardApp: App {
    let container: ModelContainer
    @State private var monitor: ConnectivityMonitor

    init() {
        let container = try! ModelContainer(for: ConnectivityEvent.self)
        self.container = container
        let store = EventStore(modelContainer: container)
        _monitor = State(initialValue: ConnectivityMonitor(eventStore: store))

        }
        .modelContainer(container)
    }
}
```
