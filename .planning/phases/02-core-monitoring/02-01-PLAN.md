Create `CellGuard/Services/ConnectivityMonitor.swift` with a `@Observable final class ConnectivityMonitor`.

**Imports:** `Network`, `Observation`, `Foundation`

**Published state properties (for future UI binding):**
- `private(set) var isMonitoring: Bool = false`
- `private(set) var currentPathStatus: PathStatus = .unsatisfied`
- `private(set) var currentInterfaceType: InterfaceType = .unknown`
- `private(set) var currentRadioTechnology: String?` (populated in Plan 02)

**Internal state for classification:**
- `private var previousPathStatus: PathStatus = .unsatisfied`
- `private var previousInterfaceType: InterfaceType = .unknown`
- `private var dropStartDate: Date?`
- `private var isInitialUpdate: Bool = true` (Pitfall 1: skip first callback)
- `private var debounceTask: Task<Void, Never>?` (for 500ms debounce)
- `private var lastLocation: (latitude: Double, longitude: Double, accuracy: Double)?` (set externally, used in Plan 02)

**Dependencies:**
- `private let pathMonitor = NWPathMonitor()`
- `private let monitorQueue = DispatchQueue(label: "com.cellguard.pathmonitor")`
- `private let eventStore: EventStore`

**Initializer:** `init(eventStore: EventStore)` -- stores the EventStore reference.

**`func startMonitoring()`:**
1. Guard `!isMonitoring`, set `isMonitoring = true`, `isInitialUpdate = true`
2. Set `pathMonitor.pathUpdateHandler = { [weak self] path in ... }`
3. Inside handler: dispatch to `Task { @MainActor in self?.handlePathUpdate(path) }` to serialize on main actor
4. Call `pathMonitor.start(queue: monitorQueue)`

**`func stopMonitoring()`:**
1. `pathMonitor.cancel()`
2. Stop probe timer (nil for now, Plan 02 adds it)
3. Set `isMonitoring = false`
4. Cancel any pending debounce task

**`@MainActor private func handlePathUpdate(_ path: NWPath)`:**
1. Map path status: `let newStatus = mapPathStatus(path.status)`
2. Detect primary interface: `let newInterface = detectPrimaryInterface(path)`
3. Capture `isExpensive = path.isExpensive`, `isConstrained = path.isConstrained`
4. **Initial update guard (Pitfall 1):** If `isInitialUpdate`, set `previousPathStatus = newStatus`, `previousInterfaceType = newInterface`, `currentPathStatus = newStatus`, `currentInterfaceType = newInterface`, set `isInitialUpdate = false`, and RETURN without logging.
5. **Debounce (Pitfall 6):** Cancel existing `debounceTask`. Create new `debounceTask = Task { try? await Task.sleep(for: .milliseconds(500)); await self.processPathChange(newStatus: newStatus, newInterface: newInterface, isExpensive: isExpensive, isConstrained: isConstrained) }`

**`@MainActor private func processPathChange(newStatus:newInterface:isExpensive:isConstrained:)`:**
Apply classification logic in this priority order:

- **Case 1 -- Overt drop:** `previousPathStatus == .satisfied && (newStatus == .unsatisfied || newStatus == .requiresConnection)` --> Set `dropStartDate = Date()`, call `logEvent(type: .pathChange, status: newStatus, interface: newInterface, isExpensive: isExpensive, isConstrained: isConstrained)`

- **Case 2 -- Connectivity restored:** `(previousPathStatus == .unsatisfied || previousPathStatus == .requiresConnection) && newStatus == .satisfied` --> Calculate `dropDuration = dropStartDate.map { Date().timeIntervalSince($0) }`, set `dropStartDate = nil`, call `logEvent(type: .connectivityRestored, status: newStatus, interface: newInterface, isExpensive: isExpensive, isConstrained: isConstrained, dropDuration: dropDuration)` (DAT-02)

- **Case 3 -- Wi-Fi fallback (MON-06):** `previousInterfaceType == .cellular && newInterface == .wifi && newStatus == .satisfied` --> call `logEvent(type: .pathChange, status: newStatus, interface: newInterface, isExpensive: isExpensive, isConstrained: isConstrained)`. This captures the moment the device silently falls back from cellular to Wi-Fi.

- **Case 4 -- Other meaningful transition:** `newStatus != previousPathStatus || newInterface != previousInterfaceType` --> call `logEvent(type: .pathChange, status: newStatus, interface: newInterface, isExpensive: isExpensive, isConstrained: isConstrained)`

After classification, update: `previousPathStatus = newStatus`, `previousInterfaceType = newInterface`, `currentPathStatus = newStatus`, `currentInterfaceType = newInterface`.

**Helper -- `private func mapPathStatus(_ status: NWPath.Status) -> PathStatus`:**
```swift
switch status {
case .satisfied: return .satisfied
case .unsatisfied: return .unsatisfied
case .requiresConnection: return .requiresConnection
@unknown default: return .unsatisfied
}
```

**Helper -- `private func detectPrimaryInterface(_ path: NWPath) -> InterfaceType`:**
Check `path.availableInterfaces.first?.type` first (ordered by system preference). If `.cellular` return `.cellular`, if `.wifi` return `.wifi`, if `.wiredEthernet` return `.wiredEthernet`, if `.loopback` return `.loopback`, if `.other` return `.other`. Fallback: return `.unknown`. Note: use `availableInterfaces` (ordered array) NOT `usesInterfaceType()` to avoid Pitfall 2 (multiple interfaces returning true).

**Helper -- `private func logEvent(type:status:interface:isExpensive:isConstrained:probeLatencyMs:probeFailureReason:dropDuration:)`:**
Create a `ConnectivityEvent` using ALL parameters. For `radioTechnology` and `carrierName`, pass `nil` for now (Plan 02 adds CTTelephonyNetworkInfo). For `latitude`/`longitude`/`locationAccuracy`, use `lastLocation` tuple if set. Then call:
```swift
Task {
    try? await eventStore.insertEvent(event)
}
```

**`func updateLocation(latitude: Double, longitude: Double, accuracy: Double)`:**
Public method to update `lastLocation` tuple. Called by Phase 3 location provider.

**Important implementation notes:**
- Mark the class `@Observable` (Observation framework), NOT `ObservableObject`
- Do NOT use Combine
- The `logEvent` helper has default nil values for probeLatencyMs, probeFailureReason, and dropDuration parameters
- NWPathMonitor cannot be restarted after cancel() -- if restart is needed, a new NWPathMonitor instance must be created. Add a comment noting this limitation.
