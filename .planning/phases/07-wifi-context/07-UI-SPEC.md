# Phase 7 — UI Design Contract: Wi-Fi Context

> Visual and interaction contract for Phase 7. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds a single new field — Wi-Fi SSID — to three existing surfaces:
> EventDetailView, EventLogExport (JSON + CSV), and the privacy toggle scope.
> No new screens. No navigation changes. The contract governs placement, formatting,
> nil handling, and privacy redaction of SSID across all three surfaces.

## Component Inventory: What Phase 7 Touches

### 1. EventDetailView — New "Wi-Fi" Section

**Where:** After the existing `Section("Cellular")` block, before `Section("Probe")`.

**When to show:** Only when `event.wifiSSID != nil`. The section is conditionally
rendered — identical pattern to the existing Location and Duration sections.

**Structure:**
```
Section("Wi-Fi") {
    LabeledContent("SSID", value: event.wifiSSID ?? "—")
}
```

**Layout spec:**
- Section title: `"Wi-Fi"` — matches the casing pattern of existing sections
  (`"Event"`, `"Network"`, `"Cellular"`, `"Probe"`, `"Location"`, `"Duration"`)
- Label: `"SSID"` — plain uppercase-first, no punctuation
- Value: raw SSID string as captured, no transformation
- Nil/absent handling: section is hidden entirely when `wifiSSID == nil`
  (device was not on Wi-Fi when the event was logged)

**Placement rationale:** Wi-Fi SSID is network context, sits logically after Cellular
data. The section order becomes: Event → Network → Wi-Fi → Cellular → Probe →
Location → Duration.

### 2. EventListView — No Change

SSID does not appear in the event list rows. The list row already shows event type
and timestamp. Adding SSID would overflow the compact row layout and the value is
not relevant for scanning a list. Full SSID context belongs in the detail view only.

### 3. Export — JSON

**New CodingKey:** `wifiSSID` (camelCase, matching all other export keys)

**Encoding behavior:**
- When privacy mode off (`omitLocation == false`): include `wifiSSID` as a string
  value when non-nil, omit key entirely when nil (use `encodeIfPresent`)
- When privacy mode on (`omitLocation == true`): omit `wifiSSID` key entirely,
  same as `latitude`/`longitude`/`locationAccuracy`

**Why omit rather than null:** Existing pattern for location uses `encodeIfPresent`
when not redacting; the privacy gate suppresses the field completely rather than
writing `null`. SSID follows the same pattern for consistency.

**Export metadata envelope:** No changes needed to `ExportMetadata`. The
`locationDataIncluded` flag already signals whether sensitive fields are present.
Consider renaming in a future phase, but out of scope for Phase 7.

### 4. Export — CSV

**New column header:** `wifiSSID`

**Column position:** After `locationAccuracy`, before `dropDurationSeconds`
(keep sensitive context fields adjacent — location and Wi-Fi SSID are both
geographic/environmental data).

**Value when nil:** empty string (consistent with how nil optional fields are
rendered in CSV — no placeholder text, no quotes)

**Value when privacy mode on:** empty string (stripped, same treatment as location columns)

### 5. Privacy Toggle — Label Update

**Current label:** `"Omit location data"` (Toggle in DashboardView)

**Updated label:** `"Omit location and Wi-Fi data"` (source: memory/project_ui_feedback.md —
user explicitly flagged that the privacy toggle must cover SSID)

**Toggle behavior:** No logic change — the existing `omitLocation` AppStorage key
continues to drive both location and SSID redaction. The label change communicates
the expanded scope.

## Nil / Empty State Contract

| Surface | Condition | Display |
|---------|-----------|---------|
| EventDetailView | `wifiSSID == nil` (device on cellular only) | Section "Wi-Fi" is hidden entirely — no section header, no row |
| EventDetailView | `wifiSSID == ""` (empty string captured) | Show `LabeledContent("SSID", value: "—")` — em dash as explicit nil sentinel |
| JSON export, privacy off | `wifiSSID == nil` | Key omitted (encodeIfPresent) |
| JSON export, privacy on | any value | Key omitted |
| CSV export, privacy off | `wifiSSID == nil` | Empty cell |
| CSV export, privacy on | any value | Empty cell |

> Design principle: Never show a placeholder like "Not connected" or "N/A" in the
> detail view — just hide the section. The absence of the section is self-explanatory
> in context. An empty string edge case gets an em dash to prevent a blank value row.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Section header | `Wi-Fi` |
| Field label | `SSID` |
| Privacy toggle label (updated) | `Omit location and Wi-Fi data` |
| Empty state (no events) | Unchanged — handled by existing ContentUnavailableView in EventListView |
| Error state | No new error state introduced — SSID capture is best-effort; nil on failure is silent |
| Destructive confirmation | None — no destructive actions in this phase |

**Copywriting rationale:**
- `"Wi-Fi"` matches Apple's own capitalization (not "WiFi", not "Wifi", not "wifi").
  This is consistent with the existing `InterfaceType.displayName` which returns `"Wi-Fi"`.
- `"SSID"` is the established technical term; using "Network Name" would be less
  precise in a diagnostic tool context.
- No verbose descriptions — this is a data tool, not a consumer app. Brevity is correct.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Tap event row in EventListView | Navigate to EventDetailView (unchanged) |
| View EventDetailView with Wi-Fi event | "Wi-Fi" section visible with SSID value |
| View EventDetailView with cellular-only event | "Wi-Fi" section absent — no empty section |
| Toggle privacy switch | Label reads "Omit location and Wi-Fi data"; both location and SSID fields stripped from all export formats |
| Export JSON with privacy off | `wifiSSID` key present for events that had Wi-Fi |
| Export JSON with privacy on | `wifiSSID` key absent for all events |
| Export CSV with privacy off | `wifiSSID` column populated where available |
| Export CSV with privacy on | `wifiSSID` column empty for all rows |

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| SSID LabeledContent | Uses `LabeledContent` — VoiceOver reads label then value automatically |
| Section "Wi-Fi" | Section header is read by VoiceOver before rows — no additional work needed |
| Privacy toggle | Updated label string is read by VoiceOver — "Omit location and Wi-Fi data, toggle, off" |
| SSID values | No truncation — SSID strings are short (≤32 chars per 802.11 spec); no `.lineLimit` needed |
