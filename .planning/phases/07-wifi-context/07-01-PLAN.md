<objective>
Add Wi-Fi SSID capture to every connectivity event in CellGuard. The SSID is captured at event creation time via NEHotspotNetwork.fetchCurrent(), stored as an optional String in SwiftData, displayed in the event detail view, included in JSON export with privacy-toggle redaction, and gated behind the Access WiFi Information entitlement.

Purpose: Provides environmental context (home, office, public network) for each logged connectivity event, making it easier to correlate cellular drops with specific Wi-Fi environments in the Apple Feedback Assistant report.

Note: EventLogExport.swift requires no changes — export encoding is driven entirely by ConnectivityEvent's Codable conformance. Adding wifiSSID to the model's encode(to:) method automatically includes it in JSON output. CSV export does not exist in the codebase (WIFI-03 CSV portion deferred per RESEARCH.md open question #2).

Output: Updated ConnectivityEvent model with wifiSSID field, SSID capture in ConnectivityMonitor, Wi-Fi section in EventDetailView, privacy-gated JSON export, entitlements file, and updated privacy toggle label.
</objective>

**ConnectivityMonitor.swift -- 3 changes:**

1. **Add import** at the top of the file (after line 4 `import CoreTelephony`):

import NetworkExtension

2. **Add captureWifiSSID() helper** after the `captureCarrierName()` method (after line 346). Place it in the same MARK section as the other capture helpers:
```swift
/// Captures the current Wi-Fi SSID if available.
/// Returns nil when not on Wi-Fi, missing entitlement, or in restricted background context.
private func captureWifiSSID() async -> String? {
    let network = await NEHotspotNetwork.fetchCurrent()
    return network?.ssid
}
```

3. **Update logEvent() method** (lines 491-522). The SSID capture is async so it must be awaited inside the existing Task block. Move the ConnectivityEvent creation inside the Task so the SSID is captured before event persistence. The updated method body becomes:

```swift
private func logEvent(
    type: EventType,
    status: PathStatus,
    interface: InterfaceType,
    isExpensive: Bool,
    isConstrained: Bool,
    probeLatencyMs: Double? = nil,
    probeFailureReason: String? = nil,
    dropDuration: Double? = nil
) {
    // Capture synchronous metadata outside the Task
    let radioTech = captureRadioTechnology()
    let carrier = captureCarrierName()
    let location = lastLocation

    Task {
        let ssid = await captureWifiSSID()

        let event = ConnectivityEvent(
            eventType: type,
            pathStatus: status,
            interfaceType: interface,
            isExpensive: isExpensive,
            isConstrained: isConstrained,
            radioTechnology: radioTech,
            carrierName: carrier,
            probeLatencyMs: probeLatencyMs,
            probeFailureReason: probeFailureReason,
            latitude: location?.latitude,
            longitude: location?.longitude,
            locationAccuracy: location?.accuracy,
            dropDurationSeconds: dropDuration,
            wifiSSID: ssid
        )

        try? await eventStore.insertEvent(event)
    }

    scheduleDropNotification(eventType: type)
}
```

Key design points:
- Synchronous captures (`captureRadioTechnology()`, `captureCarrierName()`, `lastLocation`) are captured BEFORE the Task to preserve the exact state at event time.
- `captureWifiSSID()` is awaited inside the Task because it is async.
- `scheduleDropNotification` stays outside the Task -- it does not need the SSID.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSID data -> local storage | Wi-Fi SSID is sensitive location-revealing data stored in SwiftData |
| SSID data -> JSON export | SSID included in export file that leaves the device via share sheet |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | I (Information Disclosure) | ConnectivityEvent.encode(to:) | mitigate | SSID is gated behind the existing `omitLocation` privacy flag in encode(to:). When privacy toggle is on, wifiSSID is omitted from JSON output entirely -- same pattern as latitude/longitude/locationAccuracy. |
| T-07-02 | I (Information Disclosure) | EventDetailView | accept | SSID visible on screen in event detail. Acceptable because: (1) device is single-user, (2) app has no remote access, (3) user is viewing their own data. Standard iOS screen lock protects against physical access. |
| T-07-03 | I (Information Disclosure) | SwiftData local storage | accept | SSID stored unencrypted in SwiftData alongside existing location data. Acceptable because: (1) iOS Data Protection encrypts app sandbox at rest, (2) no external data transmission, (3) consistent with how latitude/longitude are already stored. |
</threat_model>
